What it is

Sotama is a rule engine on Solana.

Click a rule in the builder. Deposit funds into a PDA the program owns. A keeper watches the trigger. The keeper signs the execute transaction when the condition hits. The program enforces destination, slippage, cadence, and ownership on every fire.

No script. No server. No custody loss.

Trust model

Compromise the keeper.
Funds stay put.

01 · Ownership

PDA holds funds.

The program owns the deposit, not the keeper. The keeper signs transactions. The keeper never holds your tokens.

02 · Safety rails

Built-in checks.

Funds go only to your wallet. Only the token you picked can be sent. Prices stay inside the limit you set. Rules respect the schedule you set. The program checks every one of these before anything moves.

03 · Authority

One keeper. Set onchain.

The program accepts signatures from a single configured pubkey. Any other signer fails the check. Rotating the keeper takes the admin role, which lives on a hardware wallet.

Plus a one-way kill switch. Once flipped, no new fires. The admin runs a wind-down that refunds deposits to users.

Speaker notes
  • Lead line: compromise the keeper key, attacker still cannot redirect a single SOL.
  • Solana-specific point: this trust model uses PDAs + onchain CPI guards. EVM equivalents need more trust in the relayer.
  • Kill switch detail: one-way, refunds users, rent flows to a configured treasury. Tested on devnet.
